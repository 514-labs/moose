# Ingesting via API Endpoints

As you define Data Models in your `/datamodels` folder, Moose will automatically generate ingestion points for you
to send data to. Ingestion points are HTTP endpoints that you can send data to by making a `POST` request.

<Callout type="info">
  {" "}
  You can inspect these endpoints by running `moose ls` in your terminal. [Learn
  more](/moose-cli#list)
</Callout>

## Sending Data to the Ingestion Endpoints

You can send data to these endpoints from your application, from a script, or from a tool like `curl`. Below are some snippets for sending data to the `/ingest/UserActivity` endpopint, which Moose automatically generated for the `UserActivity` Data Model ([see the Data Models section](/building/data-models#example-data-model-in-typescript))

<Tabs items={['curl', 'javascript', 'python']}>
  <Tabs.Tab>
    ``` bash filename="Terminal" copy
  curl -X POST \
    -H "Content-Type: application/json" \
    -d '{"eventId": "1234567890", "timestamp": "2019-01-01 00:00:01", "userId": "123456", "activity": "click"}' \
    http://localhost:4000/ingest/UserActivity
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```javascript filename="Your Javascript/Typescript App or Script" copy
  fetch('http://localhost:4000/ingest/UserActivity', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      eventId: '1234567890',
      timestamp: '2019-01-01 00:00:01',
      userId: '123456',
      activity: 'click'
    })
  })
  ```
  </Tabs.Tab>
  <Tabs.Tab>
  ```python filename="Your Python App or Script" copy
    import requests

    requests.post('http://localhost:4000/ingest/UserActivity', json={
      'eventId': '1234567890',
      'timestamp': '2019-01-01 00:00:01',
      'userId': '123456',
      'activity': 'click'
    })
    ```

  </Tabs.Tab>
</Tabs>

<Callout type="warning">
  Make sure your Moose development server is running before sending data to the
  ingestion endpoints. The dev server runs on `localhost:4000` by default. You
  can start the server by running `moose dev` in your terminal. [Learn
  more](/moose-cli#dev)
</Callout>
