# Project Structure

<LanguageSwitcher />
Moose relies heavily on structural conventions to make sure you can build quickly and reliably. That means the project structure is important.

The following is the basic structure of a Moose project:

## Top-level Folders and Files

|                   |                                                                                                      |
| ----------------- | ---------------------------------------------------------------------------------------------------- |
| app               | The main folder for your application.                                                                |
| .moose            | Moose internals. Your auto-generated project documentation is stored here.                               |
| package.json      | The default [npm package manifest file](https://docs.npmjs.com/cli/v10/configuring-npm/package-json) |
| moose.config.toml | Configuration file specific to Moose for your application.                                           |

## App folders

|            |                                                                                               |
| ---------- | --------------------------------------------------------------------------------------------- |
| datamodels | The folder where you define your [Data Models](../data-models).                |
| functions  | The folder where you define your [Streaming Functions](../stream-processing) |
| blocks     | The folder where you define your [Blocks](../db-processing)                     |
| apis       | The folder where you define your [Consumption APIs](../consumption-apis)       |
