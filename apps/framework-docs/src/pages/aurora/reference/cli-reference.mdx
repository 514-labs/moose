# CLI Reference

## Install CLI

```bash filename="Terminal" copy
bash -i <(curl -fsSL https://fiveonefour.com/install.sh) aurora
```

## Commands

### init 

Creates a data engineering project with Moose, with Aurora MCP preconfigured.

```bash filename="Terminal" copy
aurora init <project-name> <template-name> <--mcp <host>> <--location <location>>
```

| Argument/Flag | Required? | Description | Example |
|--------------|-----------|-------------|---------|
| `<project-name>` | Required | Name of your application. | `my-moose-app` |
| `<template-name>` | Required (if not using `--from-remote`) | The template you are basing your application on. Use `moose templates list` to see available templates. | `typescript-empty`, `python-empty`, `ads-b` |
| `--mcp` | Not Required | Choice of which MCP host to use. | Default: `cursor-project`; possible values: `claude-desktop`, `cursor-global`, `cursor-project`, `windsurf-global` |
| `--location` | Not Required | Location of your app or service. | Default: current directory; e.g. `/my-service` |
| `--no-fail-already-exists` | Not Required | By default, the init command fails if `location` exists, to prevent accidental reruns. This flag disables the check. |  |
| `--from-remote <connection-string>` | Not Required | If you are creating a project from a pre-existing Clickhouse database, use this flag. Do not use a template. | `'clickhouse://user:pass@host:port/db'` |
| `--language <language>` | Not Required | (Only used if `--from-remote` is used) The language of your application. | `typescript` or `python` |

### connect 

Connects to an existing data source. Currently only `clickhouse` is supported.

```bash filename="Terminal" copy
aurora connect clickhouse <--connection-string <connection-string>> <--mcp <claude-desktop>>
```

| Argument/Flag | Required? | Description | Example |
|--------------|-----------|-------------|---------|
| `--connection-string` | Required | The connection string to your Clickhouse database. | `clickhouse://user:pass@host:port/db` |
| `--mcp` | Not Required | Choice of which MCP host to use. | Default: `cursor-project`; possible values: `claude-desktop`, `cursor-global`, `cursor-project`, `windsurf-global` |

### setup

Takes an existing data engineering project build with Moose and configures Aurora MCP for it.

```bash filename="Terminal" copy
aurora setup [path] <--mcp <host>>
```

| Argument/Flag | Required? | Description | Example |
|--------------|-----------|-------------|---------|
| `[path]` | Not Required | Path to the Moose project (defaults to current directory). | `.` or `./my-moose-project` |
| `--mcp` | Not Required | Choice of which MCP host to use. | Default: `cursor-project`; possible values: `claude-desktop`, `cursor-global`, `cursor-project`, `windsurf-global` |

### config

Configure Aurora settings

#### config focus

List Aurora configured projects, allows selection of project globally configured hosts are "focused" on. No more than one Moose Project may be run at a time.

```bash filename="Terminal" copy
aurora config focus
```

#### config keys

Updates all MCP files for projects listed in ~/.aurora/aurora-config.toml to use updated API key.

```bash filename="Terminal" copy
aurora config keys <KEY>
```

| Argument/Flag | Required? | Description | Example |
|--------------|-----------|-------------|---------|
| `<KEY>` | Required | Your Anthropic API key. If you don't have an Anthropic API key, see the Anthropic initial setup guide: https://docs.anthropic.com/en/docs/initial-setup | |


#### config tools

Toggles availability of experimental MCP tools. See Tools documentation for which tools are in `standard` and which are added in the `experimental` sets.

```bash filename="Terminal" copy
aurora config tools 
```

See [Tools](./tool-reference) for more information on which tools are available.
