## Step 2: Create Your Project (1 minute)

<Steps>
### Initialize your project

<TypeScript>
```bash filename="Terminal" copy
moose init my-analytics-app typescript
cd my-analytics-app
npm install
```
</TypeScript>

<Python>
```bash filename="Terminal" copy
moose init my-analytics-app python
cd my-analytics-app

# Create virtual environment (recommended)
python3 -m venv .venv
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```
</Python>

### Start your development environment

```bash filename="Terminal" copy
moose dev
```

<Callout type="success" title="✅ Success indicator">
You should see output like:
```txt
✅ ClickHouse database ready
✅ Redpanda streaming ready  
✅ HTTP APIs ready
🚀 Moose dev server running on http://localhost:4000
```
</Callout>

<Callout type="info" title="What Just Happened?">
You initialized a new Moose project and started your local development environment. All required services (ClickHouse, Redpanda, APIs) are running locally.
</Callout>

## Step 5: Hot Reload Schema Changes (1 minute)

1. Open <TypeScript inline>`app/ingest/models.ts`</TypeScript><Python inline>`app/ingest/models.py`</Python>
2. Add a new field to your data model:

<TypeScript>
```ts filename="app/ingest/models.ts" {5} copy
export interface Foo {
  id: Key<string>;
  timestamp: Date;
  data: string;
  newField: string; // Add this line
}
```
</TypeScript>

<Python>
```python filename="app/ingest/models.py" {6} copy
class Foo(BaseModel):
    id: Key[str]
    timestamp: datetime
    data: str
    new_field: str  # Add this line
```
</Python>

3. Save the file and watch your terminal

<Callout type="success" title="Success indicator">
You should see Moose automatically update your infrastructure:

```txt
~ API Endpoint: Foo - Updated validation schema
~ Topic: Foo - Schema updated
```
</Callout>

## Need Help?

<ToggleBlock openText="Common Issues & Solutions" closeText="Hide troubleshooting">
<Tabs items={["Docker Issues", "Version Issues", "Port Conflicts", "Permission Issues"]}>
<Tabs.Tab>
**Docker not running:**
```bash filename="Terminal" copy
# macOS
open -a Docker

# Linux
sudo systemctl start docker

# Verify Docker is running
docker ps
```

**Docker out of space:**
```bash filename="Terminal" copy
docker system prune -a
```
</Tabs.Tab>

<Tabs.Tab>
<TypeScript>
**Node.js version too old:**
```bash filename="Terminal" copy
# Check version
node -v

# Install Node 20+ with nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 20
nvm use 20
```
</TypeScript>

<Python>
**Python version too old:**
```bash filename="Terminal" copy
# Check version  
python3 --version

# Install Python 3.12+ with pyenv
curl https://pyenv.run | bash
pyenv install 3.12
pyenv local 3.12
```
</Tabs.Tab>

<Tabs.Tab>
**Port 4000 already in use:**
```bash filename="Terminal" copy
# Find what's using port 4000
lsof -i :4000

# Kill the process (replace PID)
kill -9 <PID>

# Or use a different port
moose dev --port 4001
```
</Tabs.Tab>

<Tabs.Tab>
**Permission denied:**
```bash filename="Terminal" copy
# Fix Docker permissions (Linux)
sudo usermod -aG docker $USER
newgrp docker

# Fix file permissions
chmod +x ~/.moose/bin/moose
```
</Tabs.Tab>
</Tabs>
