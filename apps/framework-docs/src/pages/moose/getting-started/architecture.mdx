## What's Included with Moose

Moose provides a complete analytics backend and developer toolkit out-of-the-box:

- **Moose Core (Orchestrator):** The engine that manages all infrastructure, APIs, and hot reloading. You don't interact with Moose Core directly—instead, you write code using the Moose libraries.
- **Moose Libraries:**  
  - [`py-moose-lib`](../../packages/py-moose-lib): Python SDK for defining pipelines, schemas, workflows, and APIs.
  - [`ts-moose-lib`](../../packages/ts-moose-lib): TypeScript SDK for the same, with type safety.
- **ClickHouse:** High-performance, columnar analytical database for storage and fast queries.
- **Redpanda/Kafka:** Built-in event streaming for real-time data pipelines.
- **Temporal:** Workflow engine for background jobs and orchestration.
- **Built-in HTTP APIs:** For data ingestion and consumption, auto-generated from your code.
- **Hot reload & local dev:** All infrastructure runs locally with a single command (`moose dev`).
- **Monitoring & Metrics:** Built-in metrics endpoints for observability.
- **Dead Letter Queues:** Automatic handling of failed events.

> **How it fits together:**  
> You write code using the Moose libraries (`py-moose-lib` or `ts-moose-lib`). Moose Core reads your code, manages the infrastructure, and hot-reloads changes as you develop. You interact with Moose by writing code—not by manually configuring servers or services.

## Moose Core Layer

Moose Core is the engine that powers all Moose analytics infrastructure. It provides:

- **Rust HTTP Endpoints for Ingest:** High-performance endpoints with schema validation, routing incoming data into streaming topics.
- **Streaming Consumer:** Continuously loads data from streaming topics into ClickHouse for durable, queryable storage.
- **Streaming Functions:** Orchestrates Python/Node.js lambda functions that consume from topics and transform data in-flight as it streams through the system.
- **Consumption APIs:** Type-safe HTTP endpoints that connect to ClickHouse and expose analytical data to consumers and applications.
- **Process Management & State Tracking:** Moose Core manages all these processes, tracks the internal state of tables, topics, and APIs, and computes diffs between your code and the current infrastructure. This enables hot reloading of pipelines, schemas, functions, and queries as you develop—no manual migrations or restarts required.
