# Initializing a Data Model

<LanguageSwitcher />

## Using the CLI

To create a new Data Model in your Moose application, use the `moose data-model init` command. This will generate a <TypeScript>`.ts`</TypeScript> <Python>`.py`</Python> file in the `/datamodels` directory and define your Data Model within it.

For detailed information about the `moose data-model init` command, you can run the following in your terminal:

<TypeScript>
```bash filename="Terminal" copy 
npx moose-cli data-model init --help 
```
</TypeScript>

<Python>
```bash filename="Terminal" copy 
moose-cli data-model init --help 
```
</Python>

<Callout type="info">
  If you have sample data in a `.json` or `.csv` file, you can use the
  `--sample` flag to specify the path to that file. The CLI will then infer the
  Data Model schema from the provided file and create the Data Model for you.
</Callout>

## Creating a new Data Model Manually

There are two additional ways to initialize a new Data Model in your Moose project:

- By creating a new <TypeScript>`.ts`</TypeScript> <Python>`.py`</Python> file in the `/datamodels` directory of your Moose application and defining a new <TypeScript>interface</TypeScript> <Python>dataclass</Python> inside this file.
- By appending a new interface or dataclass to an existing <TypeScript>`.ts`</TypeScript> <Python>`.py`</Python> file in your `/datamodels` directory
